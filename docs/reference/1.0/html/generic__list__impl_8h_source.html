363 /*!
364  * \brief Create instance of the GenericList from the generator method (generator-based generic list)
365  *
366  * List generator method should follow the function signature: nonstd::optional<Value>() . Non-empty optional returned from the generator means that generated
367  * list isn't empty yet. The first returned empty optional object means the end of the generated sequence. For instance:
368  * ```
369  * jinja2::MakeGenericList([cur = 10]() mutable -> nonstd::optional<Value> {
370  * if (cur > 90)
371  * return nonstd::optional<Value>();
372  *
373  * auto tmp = cur;
374  * cur += 10;
375  * return Value(tmp);
376 });
377  * ```
378  * This generator produces the following list: `[10, 20, 30, 40, 50, 60, 70, 80, 90]`
379  *
380  * @param fn Generator of the items sequences

382  */
