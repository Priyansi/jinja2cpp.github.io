40  * \brief Generic list enumerator interface
41  *
42  * This interface should be implemented by the lists of any type. Interface is used to enumerate the list contents item by item.
43  *
44  * Implementation notes: Initial state of the enumerator should be "before the first item". So, the first call of \ref MoveNext method either moves the
45  * enumerator to the first element end returns `true` or moves enumerator to the end and returns `false` in case of empty list. Each call of \ref GetCurrent
46  * method should return the current enumerable item.
47  */
48 struct ListEnumerator
49 {
50  //! Destructor
51  virtual ~ListEnumerator() = default;
52 
53  /*!
54  * \brief Method is called to reset enumerator to the initial state ('before the first element') if applicable.
55  *
56  * For the sequences which allow multi-pass iteration this method should reset enumerator to the initial state. For the single-pass sequences method
57  * can do nothing.
58  */
59  virtual void Reset() = 0;
60 
61  /*!
62  * \brief Method is called to move the enumerator to the next item (if any)

64  * @return `true` if enumerator successfully moved and `false` if enumerator reaches the end of the sequence.
65  */
66  virtual bool MoveNext() = 0;
67

71  * @return Value of the item if the current item is valid item and empty value otherwise

73  virtual Value GetCurrent() const = 0;
74

80  virtual ListEnumeratorPtr Clone() const = 0;

93 /*!
94  * \brief Generic list implementation interface
95  *
96  * Every list implementation should implement this interface for providing access to the items of the list. There are several types of lists and implementation
97  * notes for every of them:
98  * - **Single-pass sequences** . For instance, input-stream iterators, generator-based sequences. This type of generic lists should provide no size and no indexer. Enumerator of such sequence supports should be moveable but non-copyable and non-resetable.
99  * - **Forward sequences** . For instance, single-linked lists. This type of lists should provide no size and indexer. Enumerator should be moveable, copyable and resetable.
100  * - **Bidirectional sequences**. Have got the same implementation requirements as forward sequences.
101  * - **Random-access sequences**. Such as arrays or vectors. Should provide valid size (number of stored items), valid indexer implementation and moveable,
102  * copyable and resetable enumerator.
103  *
104  * It's assumed that indexer interface is a part of list implementation.
105  */
106 struct ListItemAccessor
107 {
